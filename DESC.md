## 1. 문제 정의

### 1-1. 시장 포화와 발견 가능성(Discoverability) 위기

**[context]**
스팀에 등록되는 신규 게임 수는 해마다 가파르게 증가하고 있으며, 유저의 관심은 유한하기 때문에 개별 게임이 발견될 확률은 구조적으로 낮아지고 있다.

**[data]**

- 스팀 연간 신규 출시 수 (2024): 18,506개 — 출처: Randomgamegenerator
- 스팀 연간 신규 출시 수 (2025): 20,017개 — 출처: Randomgamegenerator
- 위시리스트→구매 전환율 (2024): 0.125 — 출처: Mordor Intelligence
- 수익 편중: 상위 0.5% 미만 게임이 전체 수익의 80% 차지 — 출처: Medium
- 대상 모수: 스팀 인디게임 약 12,000개 (2024 기준) — 출처: Medium

**[insight]**
좋은 게임을 만들더라도 99.5%의 게임은 재정적 생존이 불가능한 수준에 놓여 있다. 게임의 품질과 시장 성과 사이에 구조적 단절이 존재한다.

### 1-2. 마케팅 비용 급등과 개발자의 구조적 한계

**[context]**
인디 개발자 대부분은 자기자금으로 개발하며, 별도 마케팅 예산을 확보하는 것 자체가 어려운 구조이다. 그럼에도 마케팅 비용은 빠르게 오르고 있어 시간과 비용 양쪽에서 개발 자체를 압박하고 있다.

**[data]**

- 자기자금 개발 비율: 56% — 출처: GDC 2025 설문 (응답자 3,000명+)
- 크라우드펀딩 활용 비율: 11% — 출처: GDC 2025 설문
- 고객 획득 비용(CAC) 변화: 2년 전 대비 약 2배 증가 — 출처: Mordor Intelligence
- 마케팅 비용 비중 (현재): 프로젝트 전체 지출의 30~50% — 출처: Mordor Intelligence
- 마케팅 비용 비중 (과거): 프로젝트 전체 지출의 10~20% — 출처: Mordor Intelligence
- 마케팅이 소모하는 개발 시간: 전체의 약 40% — 출처: WiFiTalents

**[insight]**
마케팅 예산도, 마케팅에 쓸 시간도 부족한 상태에서 비용은 계속 오르고 있다. 인디 개발자에게 마케팅은 "해야 하지만 할 수 없는" 영역이다.

### 1-3. 퍼블리셔 유무에 따른 생존 격차

**[context]**
퍼블리셔의 마케팅 역량은 수익에 직접적 영향을 주지만, 퍼블리셔 계약은 높은 수익 배분, 마케팅 결정권 이양 등의 대가를 수반한다. 자기자금 개발자(56%)에게 이러한 조건은 감당하기 어렵다.

**[data]**

- 퍼블리셔 계약 게임 수익 중간값: $16,222 — 출처: Substack
- 독자 출시 게임 수익 중간값: $3,285 — 출처: Substack
- 수익 격차: 약 5배

**[insight]**
퍼블리셔 없이 마케팅을 해야 하지만 그 방법을 모르는 것이 핵심 딜레마이다. 이 격차를 기술로 줄이는 것이 Gametopia의 존재 이유이다.

### 1-4. 놓치고 있는 기회 — 스트리머·유튜버 홍보 채널

**[context]**
스트리머/유튜버를 통한 노출은 인디 게임에게 비용 대비 효과가 가장 높은 마케팅 채널 중 하나이다. 그러나 개발자와 스트리머 사이에 효율적인 연결 통로가 없어 양쪽 모두에게 니즈가 있지만 매칭이 이루어지지 않는 시장 실패 상태이다.

**[data]**

- YouTube를 통해 게임을 발견하는 게이머 비율: 64% — 출처: IQFluence (2024 설문)
- 평균 게임 발견 채널 수: 4~5개 — 출처: IQFluence
- 인플루언서 추천이 신규 게임 시도 계기가 되는 비율: 22% — 출처: Big Games Machine (2024 설문)

**[insight]**
개발자는 어떤 스트리머에게 연락해야 할지 모르고, 스트리머는 자신의 콘텐츠에 맞는 새 게임을 찾기 어렵다. 이 매칭 문제를 AI로 해결하면 양쪽 모두에게 가치를 제공할 수 있다.

## 2. 솔루션 개요

### 정의

Gametopia는 AI 기반 인디 게임 마케팅 어시스턴트와 스트리머 매칭 플랫폼을 결합한 서비스이다.

### 핵심 가치

| ID   | 가치          | 설명                                                          |
| ---- | ------------- | ------------------------------------------------------------- |
| V-01 | 마케팅 민주화 | 마케팅을 모르는 개발자도 데이터 기반 전략을 세울 수 있게 한다 |
| V-02 | 상호 연결     | 스트리머와 게임의 자연스러운 연결로 상호 윈윈 구조를 형성한다 |
| V-03 | 생태계 기여   | 인디 게임 생태계의 성장과 데이터 축적에 기여한다              |

### 서비스 구성 (투 트랙)

| 트랙 ID | 이름                 | 대상             | 핵심 기능                                                           | 해결하는 문제 |
| ------- | -------------------- | ---------------- | ------------------------------------------------------------------- | ------------- |
| TRACK-A | AI 마케팅 어시스턴트 | 인디 게임 개발자 | 스팀 게임 자동 분석, 유사 성공 게임 벤치마킹, 맞춤 마케팅 전략 추천 | 1-1, 1-2, 1-3 |
| TRACK-B | 스트리머 매칭 플랫폼 | 스트리머/유튜버  | 원하는 콘텐츠를 프롬프트로 설명하면 DB에서 적합한 게임을 추천       | 1-4           |

### 트랙 간 관계

두 트랙은 하나의 게임 데이터베이스(FEAT-03)를 공유한다. 개발자가 TRACK-A를 통해 게임을 등록·분석하면, 해당 게임은 TRACK-B의 스트리머 매칭 풀에 자동 포함된다. 단일 등록으로 분석과 홍보가 동시에 이루어지는 연결 구조이다.

## 3. 타겟 사용자

### 주 타겟: 인디 게임 개발자 (1인 ~ 소규모 팀)

| 속성        | 설명                                        |
| ----------- | ------------------------------------------- |
| 규모        | 1인 개발자 ~ 소규모 팀 (약 10명 이하)       |
| 출시 상태   | 스팀에 게임을 출시했거나 출시 예정          |

| 속성      | 설명                                                            |

## 4. 핵심 기능 상세

### FEAT-01. AI 게임 분석 & 마케팅 추천

- **트랙**: TRACK-A (핵심)
- **입력**: 스팀 게임 URL 또는 스팀 앱 ID

**분석 항목**:

| 항목        | 내용                                         |
| ----------- | -------------------------------------------- |
| 기본 정보   | 장르, 태그, 가격대, 출시일                   |
| 페이지 품질 | 스크린샷, 트레일러, 설명문 구성 및 품질 평가 |
| 유저 반응   | 리뷰 키워드 분석 (긍정/부정 요인 추출)       |
| 시장 포지션 | 경쟁 게임 대비 포지셔닝                      |

**출력 — AI 마케팅 리포트**:

| 항목        | 내용                                                                     |
| ----------- | ------------------------------------------------------------------------ |
| 벤치마킹    | 유사 장르 성공 게임의 마케팅 전략 분석 (채널, 시점, 메시지)              |
| 채널 추천   | 현재 게임에 적합한 마케팅 채널 (SNS, 커뮤니티, 스트리머, 할인 이벤트 등) |
| 페이지 개선 | 태그 최적화, 설명문 개선, 스크린샷 구성 제안                             |
| 액션 플랜   | 출시 후 타임라인별 마케팅 액션 플랜                                      |

### FEAT-02. 스팀 연동 자동 분석

- **트랙**: TRACK-A (지원)
- **의존**: FEAT-01의 입력 자동화

| 기능      | 설명                                                  |
| --------- | ----------------------------------------------------- |
| 자동 수집 | 스팀 API를 통해 게임 정보 자동 수집                   |
| 간편 입력 | URL만 입력하면 분석 시작 (수동 입력 불필요)           |
| 변화 추적 | 리뷰, 위시리스트 추이 등 주기적 변화 추적 (추후 기능) |

### FEAT-03. 성공 사례 DB

- **트랙**: TRACK-A, TRACK-B 공유
- **역할**: AI 추천의 근거 자료이자 스트리머 매칭의 게임 풀

| 기능      | 설명                                                  |
| --------- | ----------------------------------------------------- |
| 수집·분류 | 인디 게임 중 마케팅으로 성공한 사례를 체계적으로 수집 |
| 검색      | 장르별, 예산별, 홍보 채널별로 검색 가능               |
| 확장      | 지속적으로 데이터 추가 및 자동 수집으로 전환          |

### FEAT-04. 스트리머 매칭 플랫폼

- **트랙**: TRACK-B (핵심)
- **의존**: FEAT-03 (게임 DB)

**스트리머 측 흐름**:

| 단계 | 행동                                                                      |
| ---- | ------------------------------------------------------------------------- |
| 1    | 원하는 콘텐츠를 자연어로 설명 (예: "2인 협동 공포 게임, 1시간 내 클리어") |
| 2    | AI가 프롬프트를 분석하여 DB에서 조건에 맞는 게임 추천                     |
| 3    | 게임 상세 정보 (스팀 페이지, 트레일러, 핵심 특징) 제공                    |
| 4    | 관심 게임에 "플레이 의향" 표시 → 개발자에게 알림                          |

**개발자 측 연동**:

| 기능      | 설명                                      |
| --------- | ----------------------------------------- |
| 자동 등록 | 게임 등록 시 스트리머 매칭 풀에 자동 포함 |
| 알림 수신 | 스트리머의 플레이 의향 알림               |
| 키 발송   | 스트리머 리뷰 키 발송 지원 (추후 기능)    |

## 5. MVP 로드맵

| Phase   | 기간    | 목표                 | 포함 기능                           | 핵심 지표                           |
| ------- | ------- | -------------------- | ----------------------------------- | ----------------------------------- |
| Phase 1 | 0~3개월 | 핵심 가치 검증       | FEAT-01 기본, FEAT-03 수동 수집     | 리포트 생성 ≥ 100건, 재방문율 ≥ 30% |
| Phase 2 | 3~6개월 | 자동화·데이터 확장   | FEAT-02, FEAT-03 구조화·검색        | DB 게임 ≥ 500건, 개발자 만족도      |
| Phase 3 | 6~9개월 | 스트리머 매칭 생태계 | FEAT-04, 매칭 알림                  | 매칭 ≥ 월 50건, 방송 전환율 ≥ 20%   |
| Phase 4 | 9개월~  | 생태계 확장          | 대시보드, 키 발송, 커뮤니티, 다국어 | 글로벌 유저 수, 매출 변화 추적      |

### Phase 상세

**Phase 1 — 핵심 가치 검증 (0~3개월)**

- 스팀 URL 입력 → AI 마케팅 리포트 생성 기능 구현
- 기본 성공 사례 데이터 수집 (수동 + 반자동)
- 웹 기반 MVP 배포 및 인디 개발자 커뮤니티 피드백 수집

**Phase 2 — 자동화 및 데이터 확장 (3~6개월)**

- 스팀 API 연동으로 게임 정보 자동 수집
- 성공 사례 DB 구조화 및 검색 기능 구현
- 리포트 품질 고도화 (구체적 액션 아이템, 예산별 전략)

**Phase 3 — 스트리머 매칭 플랫폼 (6~9개월)**

- 스트리머 전용 페이지 구현 (프롬프트 기반 게임 추천)
- 개발자 게임 등록 → 스트리머 매칭 풀 자동 연동
- 매칭 알림 시스템

**Phase 4 — 생태계 확장 (9개월~)**

- 위시리스트·리뷰 추이 추적 대시보드
- 스트리머 리뷰 키 발송 자동화
- 커뮤니티 기능 (개발자 간 마케팅 노하우 공유)
- 글로벌 확장 (다국어 지원)

## 6. 수익 모델

> **현 단계 우선순위**: 수익 창출보다 **데이터 축적**과 **인디 게임 생태계 기여**에 있다. 수익 모델은 서비스가 충분한 가치를 증명한 후 본격 적용한다.

### 기본 방향: 프리미엄 모델

| 티어               | 대상            | 내용                                                    | 가격        |
| ------------------ | --------------- | ------------------------------------------------------- | ----------- |
| 무료               | 개발자/스트리머 | 기본 AI 분석 리포트 (요약), 스트리머 매칭 풀 등록       | ₩0          |
| 유료 A — 건당 과금 | 개발자          | 상세 마케팅 리포트 (액션 플랜, 벤치마킹 포함)           | 건당 미정   |
| 유료 B — 구독제    | 개발자          | 무제한 리포트 + 추이 추적 대시보드 + 우선 스트리머 매칭 | 월정액 미정 |

### 장기 수익 가능성
