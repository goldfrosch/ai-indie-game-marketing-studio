# 리뷰 분석

이 스킬은 게임의 유저 리뷰를 분석하여 긍정/부정 요인을 추출하고 개선 방향을 제시합니다.

## 사용 방법

```
/review-analyze <스팀_URL_또는_앱_ID>
```

## 예시

```
/review-analyze https://store.steampowered.com/app/1245620/
/review-analyze 1245620
/review-analyze
```

## 분석 항목

- **긍정 요인**: 사용자가 좋아하는 점
- **부정 요인**: 사용자가 불만족하는 점
- **패턴 분석**: 공통적으로 언급되는 주제
- **개선 제안**: 리뷰 기반 개선 방향
- **마케팅 반영**: 마케팅에 활용할 포인트

## 프롬프트

사용자가 게임 리뷰를 분석하고 싶어합니다. 다음 단계를 따르세요:

1. 사용자가 스팀 URL 또는 앱 ID를 제공했는지 확인하세요. 제공하지 않았다면 묻습니다.

2. WebFetch를 사용하여 해당 게임의 리뷰 페이지를 가져오세요.

3. 리뷰를 다음 기준으로 분석하세요:

### 긍정 요인 (Positive)
- 게임플레이, 스토리, 그래픽, 사운드 등
- 가장 자주 언급되는 칭찬 요인
- 예시 리뷰 인용

### 부정 요인 (Negative)
- 버그, 난이도, 최적화, 컨텐츠 부족 등
- 가장 자주 언급되는 불만 요인
- 예시 리뷰 인용

### 패턴 분석
- 공통 키워드 빈도
- 긍정/부정 비율 추이
- 시간별 트렌드 변화

4. 다음으로 실질적인 인사이트를 제공하세요:

### 개발자를 위한 제안
- 개선이 필요한 부분 우선순위
- 버그/이슈 해결 방향

### 마케팅을 위한 제안
- 리뷰에서 발견한 강점을 강조할 방법
- "어떤 게이머에게 어필하는지" 명확화

5. 분석 결과를 구조화된 형식으로 출력하세요.

## 참고

이 스킬은 FEAT-01의 "유저 반응: 리뷰 키워드 분석" 기능을 지원합니다.
