# 스팀 태그 최적화

이 스킬은 스팀 스토어의 태그 시스템을 최적화하여 게임의 검색 노출을 높입니다.

## 사용 방법

```
/tag-optimize <게임_장르_및_특징>
```

## 예시

```
/tag-optimize 2인 협동 공포 게임, 3D, 픽셀아트, 퍼즐
/tag-optimize 로그라이크, 탄막 슈팅, 레트로, 어려움
/tag-optimize
```

## 분석 기준

- **검색량**: 자주 검색되는 태그 우선
- **경쟁도**: 너무 많은 게임이 사용하는 태그는 피할 필요
- **정확도**: 게임의 정체성을 올바르게 표현
- **조합**: 태그 조합에 따른 노출 효과

## 프롬프트

사용자가 스팀 태그를 최적화하고 싶어합니다. 다음 단계를 따르세요:

1. 사용자가 게임의 장르와 특징을 설명했는지 확인하세요. 충분한 정보가 없다면 다음을 묻습니다:
   - 게임의 주요 장르는 무엇인가?
   - 플레이 스타일은? (액션, 전략, 퍼즐 등)
   - 비주얼 스타일은?
   - 타겟 오디언스는?

2. 다음 카테고리로 태그를 추천하세요:

### 핵심 태그 (Essential, 5~8개)
게임의 핵심 정체성을 나타내는 태그
- 장르 태그
- 플레이 스타일 태그
- 스토리/세계관 태그

### 지원 태그 (Supporting, 3~5개)
핵심 태그를 보완하는 태그
- 세부 기능 태그
- 분위기/톤 태그
- 기술적 특징 태그

### 마케팅 태그 (Marketing, 2~3개)
검색 노출을 높이는 태그
- 트렌드 태그
- 관련 장르 태그
- 캠페인 관련 태그

### 피해야 할 태그 (Avoid)
- 모호하거나 부정확한 태그
- 과도하게 경쟁이 치열한 태그
- 게임과 무관한 태그

3. 각 태그에 대해 다음을 설명하세요:
- 태그명 (영문/한글)
- 선택 이유
- 검색 노출 효과 예상

4. SteamDB 또는 유사 서비스의 태그 통계를 참고하여 근거를 제시하세요.

## 스팀 태그 제한

- 최대 20개 태그 가능
- 앱이 출시된 후 사용자가 직접 태그 추가 가능
- 개발자가 직접 태그를 추가하는 것은 출시 전에만 가능
